# UI Style Guide

Principi UI da seguire quando si estende la SPA.

## Obiettivi

- velocita operativa
- leggibilita alta
- componenti coerenti
- feedback chiari nei flussi di data entry

## Principi visuali

- UI sobria e orientata al contenuto
- contrasto sufficiente
- pulsanti e campi evidenti nei flussi core
- evitare pattern decorativi che rallentano il lavoro

## Pattern ricorrenti

- errori inline per campi
- feedback pagina per loading, empty state, error state
- conferma su azioni distruttive
- pannello totali sempre leggibile

## Dove applicarlo

- nuove pagine in `resources/js/pages/`
- componenti condivisi in `resources/js/components/`

## Nota

Questo documento e guida di stile, non fonte primaria di regole business. Per comportamento applicativo usare `docs/product/`.
